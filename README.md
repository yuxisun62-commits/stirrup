## 🛠️ Using the interface

Stirrup provides three ways to manage your work. You can use the visual interface, the command line, or your own code.

For most users, the visual interface is the best choice. This view lets you drag and drop icons to create a sequence. Each icon represents a task. You draw lines between icons to show the flow of work. 

When you add an AI node, you tell the system what task to perform. You might ask it to summarize a document, extract names from a list, or categorize emails. The engine connects these tasks and handles the inputs and outputs automatically.

## ☁️ Connecting to artificial intelligence

1. Open the settings menu inside the application.

The application encrypts this key locally on your computer. It does not share your key with anyone else.

## ❓ Frequently asked questions

Do I need to program to use this?
No. While advanced users can write code, the visual editor allows you to build workflows using only your mouse.

Is my data private?
Yes. Stirrup runs on your local machine. Your workflows and data stay on your computer, except for the information you choose to send to your selected AI provider for processing.

What if a workflow fails?
The interface highlights the specific node where the error occurred. You can click on that node to view the error log and see what went wrong. Usually, checking your internet connection or your AI key solves the problem.

Where can I find more help?
Look for the help icon in the main menu. It links to a library of guides and examples. You can also hover your mouse over any button for a brief explanation of what it does.

## 📦 Managing your projects

Your automation projects sit in a central dashboard. You can save, edit, and duplicate any flow. To export a flow, select the share option. This creates a file that you can send to other people. They can import this file into their own Stirrup application. 

The software updates automatically when you open it. This keeps your engine running on the latest version. You do not need to download the file again for minor updates.
